# Настройка переадресации

Служба переадресации позволяет перенаправлять трафик на основной сервер и используется в каскадной схеме VPN. Инструмент перенаправляет все пакеты `TCP` и `UDP` на следующих портах: `443`, `8443`, `10000:60000`. Кроме того, выполняется автоматическое перенаправление URL-адреса подписки.

:::warning
Служба переадресации работает на промежуточном сервере.

:::

## Установка службы переадресации

Служба переадресации позволяет проксировать входящие порты с промежуточного сервера на основной сервер.

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/denpiligrim/3dp-manager/main/forwarding_install.sh)
```

<small>Резюме: добавляет правила переадресации и создает службу для обновления подписки.</small>

## Удаление службы переадресации

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/denpiligrim/3dp-manager/main/forwarding_delete.sh)
```

<small>Резюме: удаляет правила и отключает службу переадресации.</small>

## Показать URL подписки

Команда для вывода текущего URL подписки из среды контейнера:

```bash
cd /opt/3dp-manager && docker compose exec node env | grep SUB_URL | cut -d'=' -f2
```

<small>Резюме: выводит статический URL подписки, который могут использовать клиенты. Работает как на основном, так и на промежуточном серверах.</small>
